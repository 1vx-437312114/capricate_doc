39 //! Basic_TMesh
40 
41 //! This class represents a manifold and oriented triangle mesh.
42 //! Vertices, Edges and Triangles are stored in the Lists V, E and T
43 //! respectively. Methods boundaries(), handles() and shells() may
44 //! be used to retrieve the respective topological entities.
45 //! Navigation of the mesh is based on the topological relationships
46 //! stored in each Vertex, Edge and Triangle.
47 //! Some methods would require a global update only to maintain
48 //! consistent values of the protected fields n_boundaries, n_handles
49 //! and n_shells. On the other hand, the same methods would work in
50 //! constant time if these values would not need to be updated.
51 //! To keep complexity as low as possible,
52 //! we make use of the 'dirty bits' d_boundaries, d_handles and
53 //! d_shells to mark that the respective entities must be updated.
54 //! The access functions boundaries(), handles() and shells()
55 //! check the status of the respective dirty bit and do a global
56 //! update (i.e., eulerUpdate()) only if necessary.
57 //! The complexity of the methods is provided as a function of a
58 //! generic 'N', which is O(V.numels()) = O(E.numels()) = O(T.numels()).
