49 //! Geometric point definition
50 
51 //! This class represents a point in the Euclidean 3D space. It can be used
52 //! to represent 3D vectors originating at (0,0,0) and terminating at the
53 //! corresponding point. Several methods of this class are intended to
54 //! manipulate vectors rather than points; for example, a call of the
55 //! method normalize is an actual normalization if the object is a vector,
56 //! but it has to be intended as a projection on the unit sphere if the
57 //! object is intended to be a point. An object of type Point is a triplet
58 //! (x,y,z) of coordinates endowed with a pointer 'info' to possible additional
59 //! information. Each coordinate is a number of type 'coord' which, by
60 //! default, is a standard double. Operations on points include addition,
61 //! subtraction, cross and dot product, and many others. This class implements
62 //! several useful operations using vector arithmethic. For example,
63 //! the simple piece of code "A = B*C;" assignes to A the value of the dot
64 //! product of B and C.
65 //! Nearly zero or nearly flat angles are automatically snapped to
66 //! exactly zero and exactly flat angles if the difference is smaller
67 //! than the global variable _acos_tolerance. This is the very basic application
68 //! of our version of the epsilon geometry for robust computation.
