Detailed Description

Optimizes the wirelength of a path by using the nearest neighbor (greedy) algorithm.

) The nearest neighbour algorithm executes quickly, but is not optimal, as it can easily miss shorter connections due to its greedy approach. It may even not find a feasible solution at all, even when one exists.

Running time: Average O(n), Worst Case O(n²)
