41 //! abstractHeap is the base class for implementing heaps.
42 //! Each implementation (class extension) must define the method
43 //! compare to be used for sorting the heap. If the objects being
44 //! sorted are non-negative numbers, a special implementation may
45 //! use the field positions to record the index of each element
46 //! within the heap. This feature is useful when there is a need to
47 //! re-sort an element whose cost changes after its insertion into
48 //! the heap. The array 'positions' must be allocated by
49 //! the extended class constructor, and must be able to contain NMAX+1
50 //! integer numbers, where NMAX is the maximum value that can be
51 //! assumed by an object.
52 
53
